# 智学助手 —— 开发顺序（AI 辅助版）

> 按顺序一步步来，每一步完成后再进入下一步。
> 每一步都标注了需要告诉 AI 的要点，方便你复制粘贴给 AI 助手。

---

## 开发策略

- **登录注册放到最后**：前 7 步不鉴权，用默认用户 ID（user_id=1）开发，最后再加登录注册和路由守卫
- **前后端分离**：React 前端 + FastAPI 后端
- **每步可运行**：每完成一步都能跑起来看到阶段性成果

---

## 第一步：项目初始化

**目标：** 创建项目骨架，能跑通 Hello World

### 后端

创建目录结构：

```
backend/
  main.py          # FastAPI 入口
  database.py      # 数据库连接
  models.py        # 数据模型
  schemas.py       # API 请求/响应格式
  config.py        # 配置（API Key 等）
  seed.py          # 预置题目数据
  routers/
    questions.py   # 题目相关
    practice.py    # 刷题相关
    wrongbook.py   # 错题本
    upload.py      # 上传题目
    dashboard.py   # 仪表盘
    qa.py          # AI 问答
    auth.py        # 🔒 登录注册（最后一步才实现）
```

安装依赖：
```
pip install fastapi uvicorn sqlalchemy python-multipart httpx python-dotenv
```

创建 `main.py`，写一个根路由返回 `{"message": "hello"}`，`uvicorn main:app --reload` 能启动。

**验证：** 浏览器访问 `http://localhost:8000` 能看到 JSON 返回

### 前端

```
npm create vite@latest frontend -- --template react-ts
cd frontend
npm install antd @ant-design/icons axios react-router-dom echarts echarts-for-react react-katex
```

把 Ant Design 中文国际化配好，写一个页面显示"Hello 智学助手"。

**验证：** `npm run dev` 能看到页面

> 告诉 AI："创建 FastAPI 后端项目骨架和 React + TypeScript 前端项目骨架"

## 第二步：数据库 + 预置题库

**目标：** SQLite 数据库建好，24 道题入库，API 能随机出题

### 后端

1. 创建 `database.py`：SQLite 连接 + 创建表
2. 创建 `models.py`：三个模型
   - **Question**：id, subject, chapter, content(含LaTeX), options(JSON), answer, knowledge_point, explanation, source, user_id(可空), review_result, created_at
   - **AnswerRecord**：id, user_id, question_id, user_answer, is_correct, error_type, error_analysis, solution_steps, learning_suggestion, similar_question, created_at
   - **User**：id, username, email, password_hash（先建表，最后一步再用）
3. 创建 `seed.py`：
   - 24 道高数选择题，覆盖 6 个知识点，每题含 LaTeX 公式
   - 启动时自动检测是否已导入
4. 创建 `routers/questions.py`：
   - `GET /questions/random`：随机返回一道题（不含答案）
   - `GET /questions/{id}`：按 ID 返回题目

### 前端

不需要改，直接用 API 测试工具或用浏览器访问接口验证。

**验证：** 调用 `GET /questions/random` 能看到题目，包含 LaTeX 公式文本

> 告诉 AI："创建 SQLite 数据库和 Question 模型，写入 24 道高数选择题 seed 数据（含 LaTeX 公式），提供随机出题 API"

## 第三步：刷题 + AI 批改核心流程

**目标：** 前端能做选择题、提交后 AI 批改、展示结果

### 重要说明

- 这一步**不鉴权**，所有操作使用 `user_id=1` 的默认用户
- 前端开发时可以手动指定默认用户

### 后端

1. 创建 `routers/practice.py`：
   - `POST /practice/submit`：提交答案
     - 接收 `{ question_id, answer }`
     - 比对正确答案
     - **答对**：调 LLM 生成肯定回复 + 解题步骤 + 知识点扩展
     - **答错**：调 LLM 生成错因分析 + 解题步骤 + 学习建议 + 相似题
     - 记录 AnswerRecord（user_id=1）
     - 返回 AI 分析结果
2. LLM 调用方式：
   - 使用兼容 OpenAI 格式的 API（DeepSeek / 硅基流动）
   - 通过 `config.py` 配置 API Key 和 Base URL
   - 回答格式要求 JSON：`{ error_type, analysis, solution_steps, suggestion, similar_question }`
3. Prompt 设计：
   - 系统 prompt 扮演数学老师
   - 输出限定 JSON 格式
   - 答对和答错不同模板

### 前端

1. 创建刷题页 `/practice`：
   - 调 `GET /questions/random` 获取题目
   - KaTeX 渲染公式
   - 显示 A/B/C/D 四个选项按钮
   - 点击选项高亮选中 → 提交按钮
   - 提交后 loading 状态
2. 创建结果页 `/result/:id`：
   - 正确/错误标识（绿色/红色）
   - 流式展示 AI 分析文本（SSE）
   - 解题步骤展示
   - 相似题展示（如果有）
   - "继续刷题"按钮

**验证：** 进入刷题页 → 做一道题 → 提交 → 看到 AI 分析流式输出

> 告诉 AI："实现刷题提交和 AI 批改功能，调用 DeepSeek API（兼容 OpenAI 格式），SSE 流式返回分析结果。当前使用 user_id=1 默认用户"

2. 创建布局组件：
   - 导航栏（仪表盘 / 刷题 / 上传题目 / 错题本）
   - 路由配置

**验证：** 先刷几道题 → 看仪表盘有数据、雷达图有颜色

> 告诉 AI："实现仪表盘页面，包含 ECharts 雷达图展示各知识点掌握度，从 AnswerRecord 实时计算"

## 第五步：错题本

**目标：** 答错的题能在错题本里查看和管理

### 后端

1. 创建 `routers/wrongbook.py`：
   - `GET /wrong-book`：错题列表（error_type != "correct"）
     - 支持筛选：`?error_type=concept_misunderstanding`
     - 支持分页
   - `GET /wrong-book/{record_id}`：单条错题详情（原题 + 分析 + 答案 + 解题步骤）

### 前端

1. 创建错题本页 `/wrong-book`：
   - 列表展示（题目摘要、错因标签、作答时间）
   - 顶部下拉筛选器（全部 / 6 种错因类型）
   - 点击跳转详情
2. 创建错题详情页 `/wrong-book/:id`：
   - 原题展示（KaTeX）
   - 用户的错误答案
   - 正确答案
   - AI 错因分析回顾
   - 解题步骤

**验证：** 先做错一道题 → 错题本能看到 → 点进去看详情

> 告诉 AI："实现错题本列表和详情页，支持按错因筛选"

## 第六步：上传题目

**目标：** 能拍照上传或手动输入选择题，AI 审查后入库

### 后端

1. 创建 `routers/upload.py`：
   - `POST /upload/image`：接收图片
     - 图片转 base64 发给 LLM
     - Prompt：识别选择题，提取题目、选项、答案
     - 判断是否为选择题 → 不是则返回错误
     - 是则返回提取结果
   - `POST /upload/manual`：接收手动输入的题目表单
   - `POST /upload/confirm`：用户确认后存入 Question 表（source="user", user_id=1）
   - 入库前 AI 审查完整性

### 前端

1. 创建上传题目页 `/upload`：
   - 两个 Tab：拍照上传 / 手动输入
   - **拍照上传**：
     - Upload 组件（支持图片）
     - 识别中 loading
     - 展示提取结果表单（可修改）
     - 非选择题 → 红色提示
   - **手动输入**：
     - 表单：题目、选项 A/B/C/D、答案、知识点
     - 支持 LaTeX
   - 确认按钮 → AI 审查 → 入库成功

**验证：** 截一道选择题图片上传 → AI 识别 → 确认入库 → 去刷题能刷到

> 告诉 AI："实现上传题目功能，支持拍照识别和手动输入两种方式，AI 审查后入库"

## 第八步：登录注册（最后再加）

**目标：** 加上用户系统，敏感 API 加上鉴权

### 后端

1. 实现 `routers/auth.py`：
   - `POST /register`：注册（bcrypt 加密密码）
   - `POST /login`：登录（返回 JWT token）
2. 写 JWT 依赖函数，加到现有路由上：
   - 所有 API 从固定 user_id=1 改为从 token 取当前用户
   - 注意：之前的所有 AnswerRecord 和 Question 都要关联到具体用户

### 前端

1. 创建登录页、注册页
2. 路由守卫：未登录跳转登录页
3. 登录后 token 存 localStorage
4. 所有 API 请求带 Authorization header

**验证：** 不登录进不了页面 → 注册 → 登录 → 正常使用所有功能

> 告诉 AI："给项目加上 JWT 登录注册功能，所有 API 加上鉴权，未登录跳转登录页"

## 第九步：部署上线

**目标：** 项目部署到线上可访问

### 方案一：前后端分离部署

1. 前端 build 后部署到 Vercel（免费）
2. 后端 Docker 部署到 Render（免费）

### 方案二：一体化部署（推荐）

1. 前端 build 后由 FastAPI 托管静态文件
2. 一个 Docker 容器包含全部
3. 部署到 Render 一个服务

> 告诉 AI："将项目部署上线，前端打包由 FastAPI 托管，部署到 Render 免费版"

## 开发建议

### 每步怎么用 AI

把每一步"告诉 AI"那句话复制给 AI 助手：
1. AI 生成代码后逐个文件创建
2. 跑起来验证（看验证标志）
3. 有问题截图发给 AI 修

### 保存进度

每完成一步 git 提交一次：
```
git add .
git commit -m "step N: 步骤名称"
```

方便哪一步坏了回退。

### API Key 配置

在 `backend/` 下创建 `.env` 文件（不要提交到 git）：
```
LLM_API_KEY=sk-your-key-here
LLM_BASE_URL=https://api.deepseek.com
LLM_MODEL=deepseek-chat
```

代码中通过 `config.py` 读取环境变量。
