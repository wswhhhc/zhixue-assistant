⚙️ 三、后端技术详解

后端采用 Python FastAPI 框架，包含 11 个路由模块、5 张数据库表、36 个 API 接口。

3.1 Web 框架

FastAPI 0.115 + Uvicorn

Python 3.11 · ASGI 异步服务器
FastAPI 自动生成 OpenAPI 文档（访问 /docs 即可看到所有接口说明），Uvicorn 作为高性能 ASGI 服务器驱动。支持异步请求处理和 SSE 流式推送。

SQLAlchemy 2.0 ORM

SQLite · 声明式映射 · 惰性加载
通过 Python 类定义数据库表结构，自动建表。复杂查询使用 func.count / func.sum / case / group_by 等在数据库层面完成统计，无需手工写 SQL。

Pydantic v2 数据校验

Rust 核心（pydantic-core）
所有 API 请求和响应都通过 Pydantic 模型定义类型，自动校验数据格式。比如 AnswerSubmit 要求 question_id 必须是 int、answer 必须是字符串，类型不对直接返回 422 错误。

🧩 五、核心功能实现

5.1 用户认证 —— JWT + bcrypt + 验证码

🔐 JWT 令牌

PyJWT · HS256 · 7天有效期
用户登录后服务端签发一个加密的 JSON Web Token，前端存在 localStorage 中，每次请求通过 Authorization 头携带。服务端通过 HTTPBearer 中间件解析验证，无状态设计无需查询数据库。

🔑 bcrypt 密码加密

bcrypt 4.2 · 12 轮迭代
密码不存明文，使用 bcrypt 算法加盐哈希后存储。校验时 bcrypt.checkpw() 对比，即使数据库泄露也无法反推密码。

🖼️ 图片验证码

Pillow · 随机噪声 · 5分钟过期
服务端用 Pillow 库实时生成验证码图片：4 个随机字符 + 4 条干扰线 + 120 个噪点，Base64 编码直接传给前端显示。UUID 标识 + 5 分钟自动过期。

5.2 AI 批改 —— SSE 流式 + 大模型

📡 SSE 流式传输

Server-Sent Events · ReadableStream
AI 批改结果不是一次性返回，而是通过 SSE 技术一个字一个字地"流"到页面上。用户提交答案后立刻看到 AI 开始"打字"，体验就像 ChatGPT 一样实时。后端用 httpx.stream() 消费 LLM 流，前端用 ReadableStream 逐帧读取。

🤖 AI 大模型调用

OpenAI 兼容 API · 硅基流动 / DeepSeek
项目对接兼容 OpenAI 格式的 LLM API，默认使用硅基流动的 Qwen3-VL-8B 模型（支持文本和图片理解）。答对和答错使用不同的提示词模板，强制输出 JSON 格式。

📋 答案分析结构

JSON 解析 · 5 类错因 · 正则清理
AI 返回的 JSON 经过三道处理：剥离 markdown 标记 → json.loads 解析 → 双反斜杠修复。结果包含：错因类型（概念偏差/计算错误/粗心/思路方向/未掌握）、分析、解题步骤、学习建议、相似题。

5.3 智能推荐 —— 掌握度算法

📊 掌握度计算

正确率 = 正确数 ÷ 总数 × 100%
系统按知识点分组统计答题记录，计算出每个知识点的掌握百分比。例如"极限与连续"做了 10 题对了 7 题，掌握度就是 70%。

🎯 薄弱点推荐

Weakest-First 策略
找出掌握度最低且有可用题目的知识点，从中随机抽一题推送给用户。没做过的知识点按 0% 掌握度处理，确保覆盖所有知识面。

5.4 拍照上传 —— 多模态 AI 识别

📸 图片题目识别

Base64 · 多模态大模型
用户上传的题目图片，服务端用 Pillow 读取后转为 Base64 格式，通过多模态 API 发送给支持图片理解的大模型（Qwen3-VL）。模型同时做 OCR 识别 + 数学语义理解 + 结构化抽取。

✅ AI 审查机制

LLM Review · 自动修正
题目入库前先让 AI 审查：题目是否完整、选项是否合理、答案是否正确。审查不通过会给出修改建议，用户可手动修改或让 AI 自动修正。

5.5 学习报告 —— ECharts 三图联动

📈 雷达图

ECharts Radar · 多边形
每个知识点作为一个维度，在一个五边形或多边形雷达图上展示各知识点的掌握百分比，直观看到哪些知识点是短板。

📉 趋势折线图

Catmull-Rom 样条插值
近 14 天正确率变化的平滑曲线，smooth: true 开启样条插值，面积填充使趋势更明显。

🥧 饼图

环形饼图 · 40%-70% 半径
错误类型分布的可视化，不同颜色区分各类错因占比，帮助了解"最容易犯什么类型的错误"。

🚀 六、部署方式

项目提供 两种运行模式：

🐳 Docker 部署

Docker Compose · 多阶段构建
双击 docker-start.bat 即可运行。Dockerfile 分两阶段：先用 node:20-alpine 构建前端（npm ci + vite build），再用 python:3.11-slim 运行后端，前端产物复制到后端的 static/ 目录。一条命令启动全部服务。

💻 本地开发

start.bat · 前后端分离运行
后端通过 Uvicorn 启动（端口 8000），前端通过 Vite 开发服务器启动（端口 5173）。后端开启 --reload 支持代码修改自动重启。

🎤 八、答辩 / 面试 话术参考

以下句子可以直接用在项目介绍或面试中，每个词都是真实的：

项目简介（30 秒版）

"我开发的智学助手是一个基于 React 19 + FastAPI 的 AI 驱动高等数学学习平台。 前端使用 TypeScript 6 和 Ant Design 6.x 企业级组件库构建交互界面， 后端基于 Python FastAPI 提供 RESTful API 和 SSE 流式服务。 项目集成了 大语言模型 API，实现了 AI 自动批改、拍照识别题目、智能推荐薄弱知识点等功能， 数据可视化使用 ECharts 6.0 展示雷达图、趋势图和错因分布饼图， 公式渲染使用 KaTeX 引擎。支持 Docker 容器化一键部署。"

技术亮点（1 分钟版）

  • SSE 流式 AI 批改：基于 Server-Sent Events 协议 + httpx.stream() + 前端 ReadableStream，实现 AI 分析结果逐字实时推送，交互体验与 ChatGPT 一致
  • 多模态题目识别：通过 OpenAI 兼容的多模态 API（Qwen3-VL），同时完成 OCR 文字识别 + 数学语义理解 + 结构化抽取三道任务
  • 自适应推荐算法：基于知识点掌握度分组的 Weakest-First 策略，动态识别薄弱环节并推送针对性练习
  • 前后端分离架构：React 19 并发模式 + Vite 8 毫秒级热更新 + FastAPI 异步 ASGI 服务 + SQLAlchemy ORM 数据层
  • 全链路安全体系：JWT 令牌鉴权 + bcrypt 密码加密 + 图片验证码防自动化 + .env 密钥隔离 + 前端路由守卫

遇到的挑战和解决

  • LLM 输出不稳定 → JSON 解析层增加 markdown 剥离 + 正则容错 + 双反斜杠修复三道保障
  • 虚拟环境激活问题 → 优化 start.bat 自动检测 venv，并提供手动分步启动替代方案
  • 数据库路径统一 → 通过 os.path.dirname 自动定位项目根目录，确保从任意目录启动都能找到数据库文件
