🔗 1. 安装 Git 必装

Git 用于从 GitHub 下载项目代码，以及后续的版本管理。

Windows

1
下载安装包
访问 Git 官网下载 Windows 版（约 60MB）。
下载 Git for Windows 官网首页
2
运行安装程序 — 全部保持默认选项，一直点"Next"即可。
3
验证安装 — 打开命令提示符（Win+R → 输入 cmd → 回车），执行： 
git --version

看到类似 git version 2.4x.x 即安装成功。

macOS

下载 Git for macOS

或使用 Homebrew 安装：brew install git

Linux (Ubuntu/Debian)

sudo apt update
sudo apt install git -y

配置 Git（首次使用）

打开命令行执行以下两行（替换为你自己的信息）：

git config --global user.name "你的名字"
git config --global user.email "你的邮箱@example.com"
💡 安装完成后需重启命令行终端，环境变量才会生效。

🐳 12. 安装 Docker Desktop（可选） 可选

Docker 提供一键容器化部署，适合不想手动配置前后端环境的用户。安装后只需双击 docker-start.bat 即可启动项目。

下载

Docker 官网下载 Windows 直接下载 macOS Intel 下载 macOS Apple Silicon 下载
1
Windows 安装 Docker Desktop — 双击安装包，
安装时会提示启用 WSL 2（Windows 虚拟化功能），点击"是"启用，安装完成后 重启电脑。
2
验证安装： 
docker --version
docker compose version

看到版本号即安装成功。Docker Desktop 启动后，右下角任务栏会显示鲸鱼图标 🐳。

系统要求（Windows）

  • Windows 10 64 位（22H2 或更新）/ Windows 11
  • BIOS 中已开启 虚拟化技术（VT-x/AMD-V）
  • 已安装 WSL 2（Docker 安装向导会自动处理）
💡 如果不确定 CPU 是否支持虚拟化，可打开任务管理器 → 性能 → CPU，查看"虚拟化"状态。
💡 Docker 启动后内存占用约 2~4GB，如果电脑内存 ≤ 8GB，建议使用本地方式启动项目。

🗺️ 6. 安装执行顺序（看完再动手）

按照以下顺序安装和配置，缺一不可：

⚙️ 8. 配置后端环境（关键步骤）
⚠️ 这是最容易出错的一步！ 请严格按照顺序操作，不要跳过任何子步骤。

8.1 创建虚拟环境

在项目根目录打开命令行：

python -m venv venv

执行后项目根目录会多出一个 venv/ 文件夹（约 20MB）。

💡 如果提示 python 不是内部或外部命令，说明 Python 未添加到 PATH。
解决方法：重新安装 Python，务必勾选 "Add Python to PATH"。

8.2 激活虚拟环境（这步经常被忘！）

Windows：

venv\Scripts\activate

macOS / Linux：

source venv/bin/activate

激活后，命令行开头会出现 (venv) 字样。
如果命令行前面没有 (venv)，说明没有激活，后续步骤会失败。

8.3 安装 Python 依赖

确保命令行前缀有 (venv)，然后执行：

cd backend
pip install -r requirements.txt

等待安装完成，看到 Successfully installed ... 即成功。

8.4 验证后端环境

确保命令行前缀有 (venv)，执行：

uvicorn --version

看到版本号即表示环境配置成功。
如果提示 uvicorn 不是内部或外部命令，说明虚拟环境未激活或依赖未安装，请返回 8.2 重新操作。

💡 安装过程中如果遇到网络慢的问题，可配置国内镜像源：
pip config set global.index-url https://mirrors.aliyun.com/pypi/simple/
🔑 9. 配置 AI 密钥和环境变量

9.1 注册 AI API 服务（任选一个）

本项目使用 AI 实现批改、问答、生成相似题等功能。你需要注册一个 API 服务并获取密钥。

🔑 注册硅基流动（推荐，有免费额度） 🔑 注册 DeepSeek

9.2 编辑 .env 配置文件

打开项目中的 backend\.env 文件，修改以下内容：

⚠️ .env 文件中原本可能带有测试密钥，请替换为你自己的 API Key，否则 AI 功能可能因额度用尽而不可用。

如果使用硅基流动（SiliconFlow）：

LLM_API_KEY=sk-你的硅基流动API密钥
LLM_BASE_URL=https://api.siliconflow.cn/v1

如果使用 DeepSeek：

LLM_API_KEY=sk-你的DeepSeekAPI密钥
LLM_BASE_URL=https://api.deepseek.com
LLM_MODEL=deepseek-chat

9.3 配置邮箱（可选，用于注册）

如果不需要注册功能（直接用测试账号），将这四行留空即可：

# SMTP 邮箱配置（注册验证码用）
SMTP_HOST=
SMTP_PORT=
SMTP_USER=
SMTP_PASS=

如果需要注册功能，填写 QQ 邮箱信息：

SMTP_HOST=smtp.qq.com
SMTP_PORT=465
SMTP_USER=你的QQ邮箱@qq.com
SMTP_PASS=你的邮箱授权码 # 不是 QQ 密码！
💡 如何获取 QQ 邮箱授权码？
登录 QQ 邮箱 → 设置 → 账户 → POP3/IMAP/SMTP 服务 → 开启并生成授权码。

9.4 配置前端 API 地址（可选）

打开 frontend/src/config.ts：

export const API_BASE = import.meta.env.VITE_API_BASE || 'http://localhost:8000'

本地开发时无需修改，默认指向后端 localhost:8000。

🚀 10. 启动项目
⚠️ 启动前请确认：
✅ 已完成 python -m venv venv（项目根目录存在 venv 文件夹）
✅ 已在虚拟环境内执行 pip install -r requirements.txt
✅ 已在 backend/.env 中填入你的 API Key
✅ 已执行过 cd frontend && npm install（前端依赖安装）

方式一：双击 start.bat 启动（适合已准备就绪的情况）

在项目根目录 双击 start.bat 即可启动。

⚠️ 关于 start.bat 的注意事项（重要！）：

① 不激活虚拟环境 — start.bat 不会自动激活 venv。
如果你之前将后端依赖安装在虚拟环境中，需要先手动激活再双击 start.bat：
venv\Scripts\activate（保持此窗口不关）→ 然后双击 start.bat
或者在启动 start.bat 之前，将后端依赖全局安装：pip install -r backend\requirements.txt

② 不安装前端依赖 — start.bat 不会执行 npm install。
如果从未安装过前端依赖，需要先执行：
cd frontend && npm install

③ 如果启动失败，请改用下方的"手动分步启动"方式排查问题。
1
双击 start.bat — 会弹出两个命令行窗口：后端（FastAPI）和前端（Vite）。
2
访问项目 — 打开浏览器访问 http://localhost:5173（前端）
或 http://localhost:8000（后端，前后端合并）。
💡 关闭两个命令行窗口即可停止服务。
💡 如果后端窗口闪退：说明 uvicorn 未找到，用上方的手动激活方式解决。
💡 如果前端窗口闪退：说明未安装 Node.js 或未执行 npm install。

方式二：手动分步启动（推荐，最稳妥）

这种方式每一步都可控，便于排查问题。

第 1 步：激活虚拟环境

venv\Scripts\activate

激活后命令行开头会出现 (venv)。

第 2 步：安装前端依赖（仅首次需要）

cd frontend
npm install

等待安装完成。

第 3 步：启动后端（新开第一个终端）

# 先激活虚拟环境
venv\Scripts\activate
# 进入 backend 目录
cd backend
# 启动后端服务
uvicorn main:app --reload --host 0.0.0.0 --port 8000

看到 Uvicorn running on http://0.0.0.0:8000 即启动成功。
如果提示 uvicorn 不是内部或外部命令：说明虚拟环境未激活或依赖未安装。

第 4 步：启动前端（另开第二个终端）

# 进入 frontend 目录
cd frontend
# 启动开发服务器
npm run dev

看到 Local: http://localhost:5173 即启动成功。
如果提示 npm 不是内部或外部命令：说明 Node.js 未安装或未添加到 PATH。

方式三：Docker 部署（适合不想手动配环境的人）

前提：已安装 Docker Desktop（第 4 节）。

在项目根目录，双击 docker-start.bat：

docker compose up -d --build

首次启动需要下载基础镜像和构建，耗时约 5~15 分钟（取决于网络速度）。
构建完成后访问 http://localhost:8000 即可使用。

停止 Docker 项目： 双击 docker-stop.bat 或执行 docker compose down。

❓ 常见问题与排查指南
问题解决方法
python 不是内部或外部命令 Python 未添加到 PATH。重新运行安装包，勾选 "Add Python to PATH"，然后重启命令行。
pip 不是内部或外部命令 同上，或使用 python -m pip 代替 pip。
npm 不是内部或外部命令 Node.js 未安装或未添加到 PATH。重新安装 Node.js。
uvicorn 不是内部或外部命令 最常见错误。 原因：虚拟环境未激活 或 依赖未安装。
解决方法：先执行 venv\Scripts\activate，确认命令行前缀出现 (venv)，
再执行 pip install -r backend\requirements.txt。
启动后端报错：端口被占用 执行 netstat -ano | findstr :8000 查看哪个进程占用了 8000 端口。
或修改启动命令中的端口号。
前端页面白屏/空白 打开浏览器开发者工具（F12）→ Console 面板，查看具体报错。
常见原因：后端未启动、API 地址配置错误。
验证码图片加载不出来 确认后端已启动且能访问 http://localhost:8000。
确认已安装 Pillow：在 venv 激活状态下执行 pip install Pillow。
AI 功能无响应 / 一直转圈 检查 backend/.env 中的 LLM_API_KEY 是否正确、是否还有余额。
如果使用硅基流动，去 控制台 查看 API Key 状态。
注册时收不到验证码 检查 SMTP 配置是否正确，或直接用测试账号 wsw / 123456 登录。
登录提示"用户名或密码错误" 确认使用正确的用户名和密码。测试账号：wsw / 123456。
如果数据库被重置，首次启动会自动初始化测试数据。
Docker 构建很慢 首次构建需要下载约 1GB 的基础镜像，属正常现象。建议使用国内镜像加速：
在 Docker Desktop 设置中配置 https://docker.mirrors.ustc.edu.cn 为镜像加速器。
