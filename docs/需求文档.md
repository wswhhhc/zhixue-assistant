# 智学助手 —— AI 驱动的学科诊断与自适应学习平台

## 需求文档 v4.0（项目实际状态）

---

## 一、项目定位

AI 学习辅助平台 Demo，核心亮点是 **"AI 自动批改 + 错因分析 + 自适应推荐"**。已实现完整功能闭环，可直接运行演示。

---

## 二、技术栈

| 层 | 方案 |
|---|------|
| 前端 | React 19 + TypeScript + Vite 8 + Ant Design 6.x |
| 图表 | ECharts（echarts-for-react） |
| 公式渲染 | KaTeX（react-katex） |
| 后端 | Python FastAPI + Uvicorn |
| ORM | SQLAlchemy |
| 数据库 | SQLite（持久化到宿主机 data/ 目录） |
| AI | SiliconFlow API（Qwen 模型，兼容 OpenAI 格式） |

## 四、功能模块

### 1. 用户系统
- 注册 / 登录 / 退出
- JWT 鉴权，路由守卫
- API 鉴权中间件

### 2. 刷题（4 种模式）

| 模式 | 功能 |
|------|------|
| 顺序答题 | 按题目 ID 顺序逐题浏览，上一题/下一题 |
| 随机答题 | 随机抽题，"换一题"跳过 |
| 智能推荐 | 分析各知识点掌握度，推荐最薄弱知识点出题 |
| 知识点练习 | 选择特定知识点，展示该知识点下所有题目，带进度和上下题切换 |

- 答案提交后 SSE 流式展示 AI 批改结果
- 每道题可收藏/取消收藏

### 3. AI 批改
- 规则判对错（选择题直接比对答案）
- **答对**：生成解题步骤 + 知识点扩展
- **答错**：错因诊断 + 解题步骤 + 学习建议 + 相似题文本
- 流式输出（SSE）
- 分析结果按类型枚举入库

### 4. AI 错因类型

| 类型 | 含义 |
|------|------|
| concept_misunderstanding | 概念理解偏差 |
| calculation_error | 计算错误 |
| careless_mistake | 审题/粗心失误 |
| wrong_direction | 思路方向错误 |
| knowledge_gap | 知识点完全未掌握 |
| correct | 回答正确 |

### 5. AI 生成相似题
- 结果页点击"AI 生成相似题"按钮
- 调用 LLM 生成一道新题（数字、表达式不同，知识点和难度相当）
- 自动入库成为正式题目
- 跳转练习新生成的题目

### 6. AI 问答
- 仪表盘顶部嵌入
- 打字问数学问题，流式回答
- 不保存历史，每次独立

### 7. 仪表盘
- **统计卡片**：今日做题数、今日正确率、总做题数、总正确率
- **打卡**：每日打卡、连续天数、近 7 天打卡日历
- **今日目标**：可设定每日目标题数，进度条展示完成度
- **学习时长**：按每题 2 分钟估算今日学习时间
- **AI 问答**：嵌入输入框
- **学习动态时间线**：近 7 天打卡 + 答题记录混合展示
- **正确率趋势图**：近 14 天正确率折线图
- **知识点掌握度**：ECharts 雷达图
- **最近错题**：最近 5 条错题，点击跳转详情

### 8. 错题本
- 分页列表展示错题摘要
- 每项支持删除
- 顶部错因类型筛选下拉框
- 点击跳转错题详情
- 顶部"重做所有错题"按钮，跳转刷题页批量重做
- 详情页：原题展示、AI 分析回顾、解题步骤、学习建议、巩固练习、重做此题、删除记录

### 9. 题目收藏
- 刷题页每道题右上角星标按钮
- 收藏页列表展示所有收藏
- 支持取消收藏和跳转练习

### 10. 上传题目
- 拍照上传：AI 识别提取题目+选项+答案
- 手动输入：表单填写
- AI 审查完整性后入库

### 11. 学习报告
- 总做题数、正确率、错误数统计
- 知识点掌握度雷达图
- 14 天正确率趋势图
- 错因类型分布饼图（剔除未识别类型）
- 薄弱知识点排行
- AI 学习建议
- 打印/导出 PDF、导出 JSON 数据

### 12. 用户设置
- 设置面板：每日目标题数
- 通过右上角齿轮图标进入

### 13. 会员系统
- 会员状态查询（剩余配额、到期时间）
- 会员兑换码兑换
- 管理员批量生成兑换码、手动设置会员
- 免费用户每日配额限制（AI问答10次、报告1次、上传3次、相似题3次）
- 配额用尽时提示升级会员

### 14. 支付系统
- 创建支付订单（支付宝/微信）
- 支付回调处理
- 支付成功后自动延长会员有效期

### 15. 管理员后台

| 模块 | 功能 |
|------|------|
| 系统概览 | 用户/题目/答题/会员数，7天趋势图、知识点正确率、题目来源分布、会员比例、行为趋势、快捷操作 |
| 用户管理 | 用户列表搜索、会员编辑（设置会员/有效期）、用户学习详情（统计数据、知识点掌握、高频错题、错误类型） |
| 题库管理 | 全部题目列表（含用户上传）、知识点筛选、搜索、编辑题目内容/选项/答案、删除题目 |
| 内容审核 | 待审核题目列表、通过/驳回、批量审核、审核后自动通知上传者 |
| 兑换码管理 | 列表查看、批量生成、启用/禁用、有效期管理 |
| 支付管理 | 支付订单列表、状态筛选（待支付/已完成/已关闭） |
| 错题分析 | 高频错题排行榜、全平台错误类型分布饼图、薄弱知识点排名 |

### 16. 通知系统
- 题目审核通过/驳回时自动通知上传者
- 用户端通知列表查看
- 标记已读功能

## 五、数据结构

```sql
User
  - id, username, email, password_hash, daily_goal(default=10)
  - membership("free"|"premium"), member_expires(datetime, nullable)

Question
  - id, subject, chapter, content(含LaTeX), options[JSON],
    answer, knowledge_point, explanation, source("system"|"user"),
    user_id(nullable), review_result, created_at

AnswerRecord
  - id, user_id, question_id, user_answer, is_correct,
    error_type, error_analysis, solution_steps, learning_suggestion,
    similar_question, created_at

Checkin
  - id, user_id, checkin_date, created_at

Favorite
  - id, user_id, question_id, created_at

UsageRecord
  - id, user_id, action("qa_ask"|"report_gen"|"upload_image"|"gen_similar"),
    date, count

MembershipCode
  - id, code(唯一), duration_days, max_uses, used_count, is_active

PaymentRecord
  - id, user_id, amount, payment_method, trade_no, status, created_at

Notification
  - id, user_id, title, content, is_read(bool), created_at
```

- 掌握度从 AnswerRecord 实时计算
- 预置题目 24 道高数选择题，覆盖 6 个知识点（极限与连续、导数与微分、不定积分与定积分、微分中值定理、多元函数、级数）
- 免费用户有每日配额限制（AI问答10次、学习报告1次、图片上传3次、相似题生成3次），会员无限制

## 六、核心流程

```
刷题页 → 选择模式 → 展示题目 → 选答案 → 提交
    ↓
后端规则判对错 → 调 LLM 生成分析（答对/答错不同 Prompt）
    ↓
SSE 流式返回 → 结果页展示
    ↓
记录 AnswerRecord → 更新掌握度
```
